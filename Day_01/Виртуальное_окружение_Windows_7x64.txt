Создание виртуального окружения. Для пользователей Windows_7x64.

На официальном сайте
https://www.python.org/downloads/
ищем строку
"Windows installer (64-bit)	Windows	Recommended	77a54a14239b6d7d0dcbe2e3a507d2f0	28217976	SIG"
скачиваем файл
"python-3.8.8-amd64.exe"
и устонавливаем python. К примеру путь установки будет:
"D:\Python"


!!!Далее в командную строку вводим команды, находясь по пути
"D:\Python"
Если находимся к примеру по пути
"C:\Users\user"
Вводим в командную строку
cd /d D:\Python

Переходим к виртуальному окружению.
Шаг_1. Установка библиотеки.

В командной строке вводим команду (без символа ">>> "):
>>> pip install virtualenv

Шаг_2. Создание виртуального окружения.

В командной строке вводим (без символа ">>> "):
>>> virtualenv venv

"venv" в данном случае – это имя окружения, оно может иметь "ЛЮБОЕ" имя.

После выполнения данной команды, в текущем каталоге будет создан новый каталог с
именем venv. Т.е. по пути "D:\Python" появится папка с именем "venv"
"venv\Scripts" – содержит скрипты для активации/деактивации окружения, интерпретатор
Python, используемый в рамках данного окружения, менеджер pip и ещё несколько
инструментов, обеспечивающих работу с пакетами Python.
"venv/lib" – каталоги, содержащие библиотечные файлы окружения. Новые пакеты будут
установлены в каталог "venv/lib/pythonX.X/site-packages"

Шаг_3. Активация виртуального окружения.

В командной строке находясь по пути с "ЛЮБЫМ" именем, например:
"D:\Python"
Для активации виртуального окружения (входа в него), в командной строке вводим (без символа ">>> "):
>>> D:\Python\venv\Scripts\activate.bat
путь может отличаться (имя пути не важно)

Если команда выполнилась успешно, то увидим, что перед приглашением в командной
строке появилась дополнительная надпись, совпадающая с именем виртуального окружения.
Путь будет выглядеть следующим образом:
"(venv) D:\Python"
путь может отличаться (зависит от пути, по которому было активировано виртуальное окружение)

Если создадим виртуальное окружение с ключем –system-site-packages:

>>> virtualenv --system-site-packages venv
то в рамках окружения venv мы будим иметь доступ к глобальному хранилищу пакетов в Windows:
\PythonXX\Lib\site-packages

Шаг_4. Деактивация виртуального окружения.

если в командной строке находимся по пути
"(venv) D:\Python\venv\Scripts"
Для деактивации виртуального окружения (выхода из него), в командной строке вводим команду (без символа ">>> "):
>>> deactivate
либо из любого другого пути (без символа ">>> "):
>>> D:\Python\venv\Scripts\deactivate.bat

Если команда выполнилась успешно, то увидим, что в имени пути исчезла приставка (venv), путь теперь выглядит так:
"D:\Python"
